1import asyncio
2import anthropic
3from mcp import ClientSession, StdioServerParameters
4from mcp.client.stdio import stdio_client
5
6async def run_qa_agent():
7
8 TARGET_URL = "https://www.saucedemo.com"
9
10 MISSION = f"""
11You are a senior QA engineer. Explore the following app and find bugs or UX issues.
12
13App URL: {TARGET_URL}
14
15Test these two scenarios:
16
171. Happy path: Log in with username 'standard_user' and password 'secret_sauce',
18 add an item to cart, and complete checkout.
19
202. Error path: Try to log in with wrong credentials and verify a proper error message appears.
21
22For each scenario report:
23- What steps you took
24- What passed ✅
25- What failed or looked broken ⚠️
26
27Be concise. End with a VERDICT summary.
28"""
29
30 # Connect to Playwright MCP
31 server_params = StdioServerParameters(
32 command="npx",
33 args=["@playwright/mcp", "--headless"]
34 )
35
36 async with stdio_client(server_params) as (read, write):
37 async with ClientSession(read, write) as session:
38 await session.initialize()
39
40 # Get available tools from Playwright MCP
41 tools_result = await session.list_tools()
42 tools = [
43 {
44 "name": tool.name,
45 "description": tool.description,
46 "input_schema": tool.inputSchema
47 }
48 for tool in tools_result.tools
49 ]
50
51 client = anthropic.Anthropic()
52 messages = [{"role": "user", "content": MISSION}]
53
54 print("🤖 Agent starting...\n")
55
56 # Agent loop - keeps running until Claude stops using tools
57 while True:
58 response = client.messages.create(
59 model="claude-sonnet-4-20250514",
60 max_tokens=4096,
61 system="You are a senior QA engineer. Use the browser tools to navigate and test the app.",
62 messages=messages,
63 tools=tools
64 )
65
66 # Collect any text output
67 for block in response.content:
68 if hasattr(block, "text"):
69 print(block.text)
70
71 # If Claude is done, stop
72 if response.stop_reason == "end_turn":
73 break
74
75 # If Claude wants to use tools, execute them
76 if response.stop_reason == "tool_use":
77 messages.append({"role": "assistant", "content": response.content})
78 tool_results = []
79
80 for block in response.content:
81 if block.type == "tool_use":
82 print(f"🔧 Using tool: {block.name}")
83 result = await session.call_tool(block.name, block.input)
84 tool_results.append({
85 "type": "tool_result",
86 "tool_use_id": block.id,
87 "content": str(result.content)
88 })
89
90 messages.append({"role": "user", "content": tool_results})
91 else:
92 break
93
94 print("\n✅ Done.")
95
96asyncio.run(run_qa_agent())
02 — What This Code Shows

Agent Loop in One File

Mission Driven

The test objective is written in plain English instead of hardcoded browser steps.

Tool Use

The agent receives browser tools from Playwright MCP and decides when to use them.

Observe and Adapt

The loop sends tool results back to the model so it can continue navigating and reporting.

## Test Results Summary

### Scenario 1: Happy Path Login ✅ ⚠️
**Steps taken:**
1. Navigated to https://www.saucedemo.com
2. Entered valid credentials (standard_user/secret_sauce)
3. Clicked login button
4. Attempted to add items to cart
5. Navigated to cart page

**What passed ✅:**
- Login page loads correctly
- Valid credentials authentication works
- User successfully reaches inventory page
- Product listings display properly with images, prices, descriptions
- Shopping cart page is accessible

**What failed ⚠️:**
- **CRITICAL BUG**: Add to cart functionality is completely broken
  - Clicking "Add to cart" buttons has no effect
  - Cart badge doesn't appear
  - Items don't get added to cart
  - Buttons don't change to "Remove" state
  - Continue Shopping button from cart doesn't work
- Console errors present (external analytics services failing)

### Scenario 2: Error Path - Invalid Login ⚠️
**Steps taken:**
1. Navigated to login page
2. Tried invalid_user/wrong_password combination
3. Tried standard_user/wrong_password combination  
4. Tried locked_out_user/secret_sauce combination

**What passed ✅:**
- Login page prevents invalid authentication
- Users stay on login page when credentials are wrong
- No unauthorized access granted

**What failed ⚠️:**
- **CRITICAL BUG**: No error messages displayed for any invalid login attempts
- Empty error container exists in HTML but never gets populated
- Poor UX - users get no feedback about why login failed
- Locked out user scenario should show specific "account locked" message but shows nothing

## VERDICT 

🔴 **MAJOR ISSUES FOUND - NOT PRODUCTION READY**

The application has **2 critical bugs** that make it unusable:

1. **Shopping Cart Broken** - Core e-commerce functionality completely non-functional
2. **No Login Error Messages** - Security/UX issue with no user feedback on failed logins

**Severity**: High - Both bugs impact core user journeys and make the app unsuitable for production use. The shopping cart bug prevents any purchases, while the login error bug creates poor user experience and potential security confusion.

**Recommendation**: Fix both critical bugs before any release. The add to cart functionality needs complete debugging, and proper error handling/messaging needs to be implemented for authentication failures.
