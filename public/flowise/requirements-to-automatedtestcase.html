Agentic AI for Automated Test Generation

This proof-of-concept (POC) project explores how an agentic AI system can automate the transformation of natural language requirements into structured test steps and test automation code for web applications. Built with Flowise, this system leverages the power of large language models to close the gap between manual QA scenarios and coded automation.

üîç Problem Statement

Manual test case authoring is time-consuming, repetitive, and prone to gaps in translation between business intent and implementation. The goal of this POC is to streamline test case generation by allowing QA engineers or product managers to input software behavior in plain English, and receive structured test steps or even executable code in return.

üß† Architecture Overview

  • Frontend Input: Users enter English-language requirements like: "The user logs in with username and password and sees the dashboard."
  • Flowise Engine: A chatflow processes the input using a Prompt Template node, a structured parser, and optionally a Custom JS function or Set Variable for debugging.
  • Output: The system returns a clean JSON object representing test steps, like:
{
  "steps": [
    { "action": "type", "target": "#username", "value": "testuser" },
    { "action": "type", "target": "#password", "value": "securepass" },
    { "action": "click", "target": "#login" },
    { "action": "wait", "target": "#dashboard" }
  ]
}

‚öôÔ∏è Flowise Configuration

The Flowise chatflow uses the following components:

  • Prompt Template: Includes clear instructions for the LLM to extract actions like click, type, wait.
  • templateFormat: Set to jinja-2 for compatibility with LLMChain processing.
  • Structured Output Parser: Parses the returned string into JSON for downstream use.
  • Set Variable / Write File Node: Optional for debugging or exporting results.

Key Learnings

  • Prompt formatting must escape all literal braces ({{ and }}) correctly to avoid runtime schema errors.
  • Temperature and model selection significantly affect test step creativity vs. precision.
  • Debugging is easiest when using Set Variable or Write File nodes before complex parsers.
